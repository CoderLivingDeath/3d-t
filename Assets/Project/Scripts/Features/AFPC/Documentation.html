AFPC Documentation

Physics-based first person character controller for Unity 6+. Modular, event-driven, fully customizable.

AFPC is a lightweight, modular first-person character controller built on Unity's physics system. Drop it into any project and get moving in minutes — or extend it with your own logic using virtual methods and UnityEvents.

MovementWalk, run, jump, double jump, air control, endurance system
LifecycleHealth, shield, damage, death, respawn, frenzy state, auto-recovery
CameraMouse look, aim zoom, speed FOV, head bobbing, camera shake
UIUI Toolkit HUD with health, shield, endurance bars and damage flash
InputUnity Input System with keyboard/mouse and gamepad support
ArchitectureSerializable classes, UnityEvents, virtual methods, custom inspector

Getting Started

  1. Create an empty GameObject and add a Rigidbody and CapsuleCollider component to it.
  2. Add the Hero component to the same GameObject.
  3. Create a Camera in the scene (not parented to the hero).
  4. Optionally create a UI GameObject with a UIDocument component and assign the AFPCUI.uxml layout, then add the AFPCUI component.
  5. In the Hero inspector under References, assign the Rigidbody, Collider, Camera, Input Actions asset, and optionally the UI.
  6. Press Play. Use WASD to move, Space to jump, Shift to run.
Note All classes live under the AFPC namespace. Add using AFPC; to your scripts when referencing them.

Hero

The Hero class is the glue that connects all systems together. It reads input via Unity's Input System, then delegates to Lifecycle, Movement, and Overview each frame.

Update Loop

CallbackSystems Called
Update()ReadInput, Looking, Aiming, Shaking, Running, Jumping, SetCrouching (camera sync), Runtime (health/shield recovery)
FixedUpdate()Crouching (collider), Accelerate (physics movement), RotateRigidbodyToLookDirection
LateUpdate()Follow (camera position)

Usage Example

using AFPC;

// Access the hero's systems from any script
Hero hero = GetComponent<Hero>();
hero.lifecycle.Damage(25);
hero.movement.BanRunning();
hero.overview.Shake(0.5f);

// Access hero info for UI
string name = hero.info.ID;
Sprite icon = hero.info.Icon;
int level = hero.info.GetInt("level", 1);

Lifecycle

Manages the health-damage-death cycle. Damage is absorbed by shield first, then health. When health reaches zero, the character dies. Health and shield recover over time at configurable intervals.

Health & Shield

PropertyDefaultDescription
referenceHealth100Maximum health. Health recovers toward this value.
healthRecoveryInterval1.0sSeconds between each +1 health tick. Set to 0 to disable.
referenceShield100Maximum shield. Shield absorbs damage before health.
shieldRecoveryInterval1.0sSeconds between each +1 shield tick. Set to 0 to disable.

When Damage(value) is called, shield is reduced first. Any overflow is applied to health. When Heal(value) is called, health is restored first, then shield.

Frenzy

Frenzy activates when health drops below a percentage threshold. Use it to trigger special effects at low health (screen effects, audio, speed boosts, etc).

PropertyDefaultDescription
frenzyThreshold20%Health percentage below which frenzy activates.

Movement

Physics-based character movement using Rigidbody velocity. Supports walking, running, jumping (with multi-jump), air control, and an endurance system that limits running and jumping.

Jumping

PropertyDefaultDescription

Crouch

Crouching shrinks the capsule collider and lowers the camera. The collider transitions smoothly over time. The character cannot stand up if there is a ceiling overhead.

PropertyDefaultDescription

crouchSpeedReduction0.5Fraction of walk speed removed while crouching. 0 = no reduction, 1 = cannot move.
crouchTransitionSpeed10.0How fast the collider and camera lerp between standing and crouching. Shared by both.
ceilingMaskDefaultLayers checked for ceiling clearance before standing up.
Collider Transition The capsule height and center lerp in FixedUpdate() so the physics simulation and collider change are always in sync, preventing camera jitter on the first frame.

Slopes

Slope handling prevents the character from walking up surfaces steeper than the slope limit and applies gravity-based sliding on steep surfaces. On walkable slopes, movement speed is optionally adjusted based on incline direction.

PropertyDefaultDescription
slopeLimit45°Maximum walkable slope angle in degrees. Surfaces steeper than this cause sliding and block uphill movement.
slopeSlideForceMul1.0Multiplier for gravity-based slide force on steep slopes. 0 = disable sliding entirely.
uphillSpeedReduction0.3Speed reduction going uphill on walkable slopes (0–1). 0 = no effect, 1 = full reduction at slopeLimit.
downhillSpeedBoost0.15Speed boost going downhill on walkable slopes (0–1). 0 = no effect, 1 = full boost at slopeLimit.
How It Works Ground detection runs every FixedUpdate using the selected groundDetectionMode. Raycast casts a single ray downward from the capsule center — immune to modular geometry seams. MultiRaycast adds a ring of rays at groundCheckRadius for better edge forgiveness while keeping seam safety. SphereCast is the most forgiving at edges but can snag on vertical faces between adjacent colliders. All modes populate the same groundHit data (normal, transform, collider) used for slope handling and platform detection. On steep slopes, gravity is projected onto the slope plane and applied as a sliding force. Input that would push the player uphill is stripped. On walkable slopes, speed is modulated based on the dot product between movement direction and the downhill vector. Jumping is blocked on steep slopes.

Camera Shake

Lens shift based shake. Requires the camera's Physical Camera mode to be enabled.

Requirement Camera shake uses camera.lensShift which requires the Physical Camera checkbox to be enabled on the Camera component.
PropertyDefaultDescription
referenceShakingAmount0Idle shake intensity. The camera continuously lens-shifts by this amount even without a Shake() call.

Crouch Camera

When the character crouches, the camera smoothly lowers to reflect the reduced eye height. The transition uses Mathf.SmoothDamp, starting from the camera's actual current position to prevent any jump on the first frame. The transition speed is shared with the collider via movement.crouchTransitionSpeed.

PropertyDefaultDescription
crouchCameraOffset-0.4How far the camera drops when crouching. Negative = down.
crouchTransitionSpeed10.0Synced from Movement at start. Controls both collider and camera lerp speed.

AFPCUI

UI Toolkit based HUD displaying health, shield, and endurance bars with a full-screen damage flash overlay. Requires a UIDocument component on the same GameObject.

Setup

  1. Create a GameObject and add a UIDocument component.
  2. Assign AFPCUI.uxml as the Source Asset and AFPC.asset as the Panel Settings.
  3. Add the AFPCUI component to the same GameObject.
  4. Drag the GameObject into the Hero's UI slot in the References panel.

The UI automatically links to the Hero on Awake and updates bar widths each frame based on current values. The damage flash triggers on any onDamage event and fades out over 0.5 seconds.

Customization

Edit AFPCUI.uxml to change bar colors, sizes, position, and labels. The script queries elements by name:

Element NamePurpose
health-fillHealth bar fill (width is set as percentage)
shield-fillShield bar fill
endurance-fillEndurance bar fill
damage-fxFull-screen red flash overlay

Custom Input

You can replace the input system entirely. The Hero class reads input into these public fields each frame:

// Movement reads from:
movement.movementInputValues   // Vector3 (x, y)
movement.runningInputValue     // bool
movement.jumpingInputValue     // bool (buffered — set true for one frame)
movement.crouchingInputValue   // bool (held state, for Hold mode)
movement.crouchingInputPressed // bool (one-frame press, for Toggle mode)

// Overview reads from:
overview.lookingInputValues    // Vector2 (x, y)
overview.lookingIsMouseDelta   // bool — true = mouse (fixed scale), false = gamepad (Time.deltaTime)
overview.aimingInputValue      // bool

Write your own input reader by setting these values directly, bypassing the Input System entirely. When providing mouse deltas, set lookingIsMouseDelta = true so that Looking() uses the correct framerate-independent scale. For gamepad or virtual stick input, leave it false.

Extensions

Extensions are modular components that plug into Hero via the extensions list and are updated each frame alongside the core systems. They are general-purpose — an extension can be anything: a movement ability, a status effect, a weapon handler, a trigger system, a runtime modifier, or any custom logic that needs access to the Hero and its subsystems.

AFPCExtension Base Class

Add an extension component to the Hero GameObject (or any child), then drag it into the Hero's Extensions list in the inspector. The order field controls execution order — lower values run first.

public class MyAbility : AFPCExtension {

    public override void Initialize () {
        // Called by Hero.Start() after all systems are ready
        // hero reference is available here
    }

    public override void OnUpdate ()      { }   // called in Hero.Update
    public override void OnFixedUpdate () { }   // called in Hero.FixedUpdate
    public override void OnLateUpdate ()  { }   // called in Hero.LateUpdate

    public override bool IsActive () => false;   // return true while ability is running; used by AFPCUI to highlight the extension row
}
hero reference The hero field is injected before Initialize() is called. Use hero.movement, hero.overview, hero.lifecycle to interact with the core systems.

To retrieve an extension at runtime:

// By type
MyAbility ability = hero.GetExtension<MyAbility>();

// By info ID
AFPCExtension ext = hero.GetExtension("grapple");
Optional Extension Pack A separate extension pack is available on the Asset Store with ready-made movement extensions: Dashing, Blinking, Wall Running, Gliding, Climbing, Ledge Grabbing, Hooking, Sliding, Stomping, and utility extensions (Vibrant, Modifying, Triggers, Slow Motion, and more).

AFPCFootsteps

Fires a step event at regular intervals while the player is grounded and moving. Step timing is distance-based — one step every N meters traveled — so it automatically adapts to walking and running speed without additional configuration. A downward raycast detects the surface under the player and passes a surface tag string to the event, keeping the extension completely audio-agnostic.

PropertyDefaultDescription
walkStepDistance2.0Meters between steps at walking speed.
runStepDistance1.4Meters between steps at running speed.
groundMaskDefaultLayers used for the downward surface raycast.
rayDistance1.5Length of the downward raycast.
  • onStep (string) — fired on each step. Parameter is the PhysicsMaterial name, the layer name if no material is set, or an empty string if nothing was detected.

Surface Detection

The raycast checks hit.collider.sharedMaterial.name first. If the collider has no physics material, it falls back to the layer name of the hit object. Hook the onStep event to a script that selects audio clips by surface name.

// Example: respond to footstep surface in another component
void OnStep (string surface) {
    switch (surface) {
        case "Concrete": PlayClip(concreteClip); break;
        case "Grass":    PlayClip(grassClip);    break;
        default:        PlayClip(defaultClip);  break;
    }
}

AFPCInteraction

Raycast-based interaction system. Each frame it casts a ray from the camera forward and checks for a component implementing IInteractable on the hit object or any of its parents. Fires focus and unfocus events as the player looks at and away from objects, and calls Interact() when the interact key is pressed.

PropertyDefaultDescription
interactAction—Input action that triggers interaction.
interactDistance2.5Maximum interaction range in world units.
interactMaskDefaultLayers considered for the interaction raycast.
  • onFocus (GameObject) — fired when the player starts looking at an interactable.
  • onUnfocus — fired when the player looks away from the focused object.
  • onInteract (GameObject) — fired on a successful interaction.
  • onInteractFailed — fired when interact is pressed but nothing is in range.

IInteractable Interface

Implement this interface on any MonoBehaviour to make it interactable:

public class Door : MonoBehaviour, IInteractable {

    public void Interact (Hero hero) {
        // Called when the player presses the interact key
        Open();
    }

    public string GetPrompt () => "Open";   // shown in UI hint
}

API

string GetFocusedPrompt ()

Returns the GetPrompt() string from the currently focused interactable, or an empty string if nothing is focused. Use this to drive interaction hint UI.

GameObject GetFocused ()

Returns the currently focused GameObject, or null.

AFPCLeaning

Hold left or right to lean — the camera rolls and shifts horizontally to simulate peeking around corners. The collider does not move; the lean is purely visual. Applied in LateUpdate after Follow() sets the base camera position, so it stacks correctly with head bobbing, crouch offset, and camera shake without interference.

PropertyDefaultDescription
leanLeftAction—Input action held to lean left.
leanRightAction—Input action held to lean right.
leanAngle15°Maximum camera roll at full lean.
leanDistance0.35Maximum horizontal camera offset at full lean.
leanSpeed10Interpolation speed toward the target lean.
  • onLeanStart — fired when the player begins leaning from neutral.
  • onLeanStop — fired when the player returns to neutral.

API

float GetLeanValue ()

Current lean value. -1 = fully left, 0 = neutral, 1 = fully right.

AFPCStepClimb

Automatically steps up small obstacles. Rigidbody-based controllers do not get free step-climbing unlike CharacterController, so without this extension the player can get caught on curbs, door frames, and small ledges. Uses three raycasts to detect a climbable step ahead — foot-level hit, clearance above, and a surface to land on — then applies a brief upward velocity boost.

PropertyDefaultDescription
maxStepHeight0.4Maximum obstacle height the player can step over.
stepUpForce4.0Upward velocity applied when a step is detected.
checkDistance0.4Forward ray length. Should roughly match the collider radius.
cooldown0.3sMinimum seconds between step boosts to prevent repeated triggering.
stepMaskDefaultLayers considered as obstacles.
  • onStep — fired each time the player steps up an obstacle.
Tuning Tips Keep maxStepHeight well below the player's collider radius to avoid false positives on ramps. If the player bounces on shallow slopes, reduce stepUpForce or increase cooldown.

AFPCMovingPlatformSupport

Tracks kinematic platforms under the player and applies their translation and rotation. When the player leaves a platform, the platform's velocity is optionally transferred. Zero-config detection — any kinematic Rigidbody on the ground layer is treated as a platform.

PropertyDefaultDescription
velocityInheritance1.0How much of the platform's velocity transfers to the player when leaving. 0 = none, 1 = full.
maxInheritedSpeed30Maximum speed that can be inherited from a platform.
  • onPlatformEnter — fired when the player lands on a moving platform.
  • onPlatformExit — fired when the player leaves a moving platform.

How it works

  1. Each FixedUpdate, reads movement.GetGroundTransform() to find what the player is standing on.
  2. If the ground transform has a kinematic Rigidbody, it is tracked as a platform.
  3. Position delta is applied via rb.MovePosition(). Yaw delta is applied via overview.RotateYaw(), including pivot-based displacement for rotating platforms.
  4. On exit (jump or walk off), platformVelocity × velocityInheritance is added to the player's velocity, clamped by maxInheritedSpeed.
  5. Platform-to-platform transfers skip velocity inheritance to prevent speed spikes.
Setup Platforms need a Rigidbody with isKinematic = true and a collider on a layer included in Movement's groundMask. Move them with MovePosition / MoveRotation in FixedUpdate for correct physics interpolation.

AFPC Wizard

An editor window that scans the open scene for Hero components and reports configuration details and errors. Open via AFPC › Wizard in the Unity menu bar.

How to use

  1. Open the window via AFPC › Wizard.
  2. The scene is scanned automatically on open.
  3. Click Scan Scene to re-scan after making changes.
  4. Click any entry to ping the corresponding GameObject in the Hierarchy.

What is checked

PanelChecks
ReferencesRigidbody, CapsuleCollider, Camera, InputActionAsset assigned. AFPCUI shown if assigned.
MovementWalk/run speeds, jump settings, endurance, crouch config, physics, ground mask. Errors if Ground Mask is Nothing or crouch height ≥ standing height.
OverviewFOV values, sensitivity (error if zero), look ranges, Speed FOV and Head Bobbing state.
LifecycleHealth and shield max and recovery rates, frenzy threshold. Error if health max ≤ 0.
ExtensionsCount, order, and enabled state of each registered extension. Error if a slot is null.

Entry colors

● GreenReference assigned or check passed.
● YellowWarning — valid but potentially misconfigured.
● RedError — missing reference or broken configuration.
● WhiteInformational value, no action required.
