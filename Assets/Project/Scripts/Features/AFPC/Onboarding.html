Developer Guide

Architecture overview and practical patterns for working with AFPC from code.

Architecture

AFPC is built around a single MonoBehaviour, Hero, that owns three subsystems.

Hero ─ the root MonoBehaviour. Owns input, orchestrates subsystems.
├─ Lifecycle health · shield · death · respawn · frenzy
├─ Movement walk · run · jump · crouch · endurance · physics
├─ Overview mouse look · FOV · head bob · camera shake
└─ AFPCUI optional HUD (health / shield / endurance bars)

The three subsystems are plain C# classes — not MonoBehaviours. They are serialized directly on Hero and initialized in Start(). They communicate outward only through UnityEvents, so you can react to any state change without modifying core files.

You interact with AFPC from your own scripts by getting a reference to Hero and calling into the subsystems: hero.lifecycle, hero.movement, hero.overview.

Hero also supports extensions — modular MonoBehaviours that plug into its update loop and have direct access to all subsystems. The base class AFPCExtension is included in the package. Both Hero and extensions carry an info field (AFPCObjectInfo) for storing display metadata and key-value data accessible at runtime. See the API Reference for details.

Scene Setup

The fastest path is to drag the included Hero prefab into your scene. For a manual setup:

  1. Create a GameObject. Add a Rigidbody, CapsuleCollider, and the Hero component.
  2. Create a child GameObject for the camera. Add a Camera component.
  3. In the Hero inspector, assign the Rigidbody, CapsuleCollider, and Camera to the Movement and Overview sections.
  4. Assign the included AFPCInput InputActionAsset to the Input Actions field.
  5. Set the Ground Mask in Movement to the layer your floor geometry is on.
  6. Press Play. The controller is ready.
Ground Mask If the character falls through the floor or never registers landing, Ground Mask is almost always the cause. It must include the layer your geometry is on and must not include the character's own layer.
Ground Detection Mode Movement offers three ground detection modes under Jumping. MultiRaycast (default) works reliably on all geometry including modular floors with seams. Raycast is a lightweight single-ray alternative. SphereCast gives the most forgiving edge detection but can snag on seams between adjacent colliders. If your levels use modular/tiled geometry, keep the default MultiRaycast.

Use the AFPC Wizard (AFPC › Wizard in the Unity menu) to scan the scene for configuration errors — missing references, invalid layer masks, zero sensitivity, and more.

Update Flow

Hero calls its subsystems in a fixed order each frame. Knowing this helps you understand when values are updated and when it is safe to read them.

PhaseWhat runs
Update Read input → check availability → mouse look → FOV (aim + speed) → camera shake → run/jump state → health and shield recovery
FixedUpdate Check availability → crouch transition → acceleration (velocity) → rigidbody rotation
LateUpdate Camera follow
Availability guard All three phases skip their work when lifecycle.Availability() returns false — i.e. when the character is dead or deactivated. If you are reading subsystem values from your own scripts and the character is dead, expect them to be stale.

Lifecycle — API

Lifecycle manages health, shield, death, and respawn. Access it via hero.lifecycle.

Methods

void Damage(float value)

Deal damage. Shield absorbs first; any remainder reduces health. Fires onShieldChanged, onHealthChanged, and onDamage. Triggers death if health reaches zero.

void Heal(float value)

Restore health up to the maximum. Fires onHeal and onHealthChanged.

void Death()

Kill the character immediately. Sets health and shield to minimum, deactivates the controller, fires onDeath.

void Respawn()

Restore the character. Resets health and shield to maximum, reactivates the controller, fires onRespawn.

void Activate() / Deactivate()

Toggle controller availability. When deactivated, Hero skips all subsystem updates. Fires onActivate / onDeactivate.

float GetHealthValue() / GetShieldValue()

Returns the current health or shield value.

bool Availability()

Returns whether the character is currently active (alive and not deactivated).

bool IsFrenzy()

Returns whether the character is in frenzy state (health below frenzyThreshold).

void AllowHealthRecovery() / BanHealthRecovery()

Enable or disable automatic health regeneration.

void AllowShieldRecovery() / BanShieldRecovery()

Enable or disable automatic shield regeneration.

void SetMaximumHealthAndShield()

Set health and shield to their maximum reference values. Called automatically by Respawn().

void SetMinimumHealthAndShield()

Set health and shield to 1. Called automatically by Death().

Lifecycle — Events

  • onDamage (float) Fired when damage is taken. Parameter: damage amount.
  • onHeal (float) Fired when healed. Parameter: heal amount.
  • onDeath Fired when health reaches zero.
  • onRespawn Fired when the character respawns.
  • onActivate Fired when the character becomes available.
  • onDeactivate Fired when the character becomes unavailable.
  • onFrenzyChanged (bool) Fired when frenzy state changes. Parameter: is frenzy active.
  • onHealthChanged (float) Fired whenever health changes. Parameter: current health.
  • onShieldChanged (float) Fired whenever shield changes. Parameter: current shield.

Lifecycle — Code Examples

Reacting to damage and death

private Hero hero;

private void Start () {
    hero = FindAnyObjectByType<Hero>();
    hero.lifecycle.onDamage.AddListener(OnDamage);
    hero.lifecycle.onDeath.AddListener(OnDeath);
    hero.lifecycle.onFrenzyChanged.AddListener(OnFrenzy);
}

private void OnDestroy () {
    hero.lifecycle.onDamage.RemoveListener(OnDamage);
    hero.lifecycle.onDeath.RemoveListener(OnDeath);
    hero.lifecycle.onFrenzyChanged.RemoveListener(OnFrenzy);
}

private void OnDamage (float amount) { /* play hit sound, flash UI */ }
private void OnDeath ()               { /* show game over screen */ }
private void OnFrenzy (bool active)   { /* enable low-health vignette */ }

Dealing damage from a hazard trigger

private void OnTriggerEnter (Collider other) {
    var hero = other.GetComponent<Hero>();
    if (hero != null) hero.lifecycle.Damage(25f);
}

Conditional respawn

private void TryRespawn () {
    if (hero.lifecycle.Availability()) return; // already alive
    hero.transform.position = spawnPoint.position;
    hero.lifecycle.Respawn();
}

Temporarily locking out recovery

// Disable shield recovery while a debuff is active
hero.lifecycle.BanShieldRecovery();

// Re-enable after the debuff expires
hero.lifecycle.AllowShieldRecovery();
Always unsubscribe If you subscribe to a UnityEvent in Start(), unsubscribe in OnDestroy(). Use named methods — not lambdas — so the listener can be removed by reference.

Movement — API

Movement handles physics, endurance, and all locomotion states. Access it via hero.movement.

State queries

bool IsGrounded()

Returns whether the character is currently on the ground.

bool IsAirborne()

Convenience inverse of IsGrounded().

Transform GetGroundTransform()

The transform the character is standing on, or null if airborne. Used by AFPCMovingPlatformSupport.

bool IsOnSteepSlope()

True when grounded on a surface steeper than slopeLimit. The character slides and cannot walk uphill or jump.

bool IsOnStableGround()

True when grounded on a walkable surface. Shorthand for IsGrounded() && !IsOnSteepSlope().

float GetGroundSlopeAngle()

Slope angle in degrees of the current ground surface. 0 when airborne.

Vector3 GetGroundNormal()

Normal of the ground surface. Vector3.up when airborne.

bool IsCrouching()

Returns whether the character is currently in the crouch state.

bool IsRunning()

Returns whether the character is currently running.

float GetEnduranceValue()

Returns the current endurance value.

Capability gating

void AllowMovement() / BanMovement(bool isStopImmediately = false)

Enable or disable walking and running. Pass true to also zero out velocity immediately.

void AllowRunning() / BanRunning()

Enable or disable sprinting.

void AllowJumping() / BanJumping()

Enable or disable jumping.

void AllowCrouching() / BanCrouching()

Enable or disable crouching. Calling BanCrouching() while crouching immediately stands the character up.

void AllowAirControl() / BanAirControl()

Ban sets airControlFactor to 0. Allow restores it. For gradual control, set movement.airControlFactor directly (0–1).

Actions

void StandUp()

Immediately restore the standing collider. Safe to call externally, e.g. on death.

void ConsumeEndurance(float amount)

Deduct endurance by amount. Fires onEnduranceDepleted if endurance reaches zero.

Movement — Events

  • onLanded Fired when the character touches the ground.
  • onAirborne Fired when the character leaves the ground.
  • onJump (int) Fired on each jump. Parameter: jump count (1 = first, 2 = double jump).
  • onRunStart Fired when the character starts running.
  • onRunStop Fired when the character stops running.
  • onEnduranceDepleted Fired when endurance reaches zero.
  • onCrouchStart Fired when the character enters crouch state.
  • onCrouchStop Fired when the character stands back up.

Movement — Code Examples

Reacting to landing (e.g. footstep sound)

private void Start () {
    hero.movement.onLanded.AddListener(OnLanded);
    hero.movement.onJump.AddListener(OnJump);
}

private void OnLanded ()           { audioSource.PlayOneShot(landClip); }
private void OnJump (int count) { audioSource.PlayOneShot(jumpClip); }

Disabling movement during a cutscene

private void StartCutscene () {
    hero.lifecycle.Deactivate();  // stops all Hero updates
}

private void EndCutscene () {
    hero.lifecycle.Activate();
}

Locking only specific capabilities

// In a menu or dialogue — allow looking but ban movement and jumping
hero.movement.BanMovement(true); // stop immediately
hero.movement.BanJumping();
hero.overview.AllowLooking();

// Restore when done
hero.movement.AllowMovement();
hero.movement.AllowJumping();

Checking movement state in your own Update

private void Update () {
    if (hero.movement.IsGrounded())  /* play walk particles */;
    if (hero.movement.IsCrouching()) /* shrink crosshair */;
    if (hero.movement.IsRunning())   /* show sprint indicator */;

    float endurance = hero.movement.GetEnduranceValue();
    enduranceBar.value = endurance / hero.movement.referenceEndurance;
}

Overview — API

Overview controls the camera: mouse look, FOV, head bobbing, and shake. Access it via hero.overview.

void HideCursor()

Lock and hide the OS cursor. Called automatically on start when hideCursor is enabled. Call manually when returning to gameplay from a menu.

void ShowCursor()

Unlock and show the OS cursor. Call when opening a pause screen or any menu that needs pointer input.

void AllowLooking() / BanLooking()

Enable or disable look input. Works with both mouse and gamepad — the active device is detected automatically.

void AllowAiming() / BanAiming()

Enable or disable the aim zoom (right mouse button FOV transition).

void Shake(float amount)

Trigger a camera shake of the given intensity. Requires Physical Camera mode on the camera. Fires onShake.

void RotateYaw(float degrees)

Add yaw rotation in degrees. Applies immediately even without mouse input. Used by AFPCMovingPlatformSupport for rotating platforms.

GameObject Search()

Fires a raycast forward from the camera using searchMask and searchDistance. Returns the hit GameObject, or null. Useful for interaction or targeting systems.

Overview — Events

  • onAimStart Fired when the player starts aiming.
  • onAimStop Fired when the player stops aiming.
  • onShake (float) Fired when the camera is shaken. Parameter: shake amount.

Overview — Code Examples

Shaking the camera on an explosion

private void OnExplosion (float intensity) {
    hero.overview.Shake(intensity);
}

Interaction system using Search

private void Update () {
    var target = hero.overview.Search();
    crosshairHighlight.SetActive(target != null);

    if (target != null && Input.GetKeyDown(KeyCode.E)) {
        target.GetComponent<IInteractable>()?.Interact();
    }
}

Reacting to aim state

private void Start () {
    hero.overview.onAimStart.AddListener(() => weaponModel.SetActive(true));
    hero.overview.onAimStop.AddListener(() => weaponModel.SetActive(false));
}

Modifying sensitivity at runtime

// e.g. from a settings slider
public void OnSensitivityChanged (float value) {
    hero.overview.sensitivity = value;
}

Cursor management for pause / menus

public void OpenPauseMenu () {
    hero.overview.ShowCursor();
    hero.overview.BanLooking();
}

public void ClosePauseMenu () {
    hero.overview.HideCursor();
    hero.overview.AllowLooking();
}

Accessing Hero

There are several ways to get a reference to the Hero component from your own scripts.

Serialized reference (preferred)

public Hero hero; // drag the Hero GameObject into this field

FindAnyObjectByType (single-player)

private void Start () {
    hero = FindAnyObjectByType<Hero>();
}

From a trigger or collision

private void OnTriggerEnter (Collider other) {
    var hero = other.GetComponentInParent<Hero>();
    if (hero == null) return;
    hero.lifecycle.Damage(10f);
}
GetComponentInParent vs GetComponent The Rigidbody and CapsuleCollider are on the Hero's root GameObject. Triggers and collisions may hit a child collider. Use GetComponentInParent<Hero>() to safely find Hero regardless of the hierarchy.

Object Info

Hero and every AFPCExtension have an info field of type AFPCObjectInfo — a serializable container for display metadata and arbitrary key-value data. Use it to feed UI elements like character select screens, ability tooltips, or HUD icons without adding custom fields to every script.

Reading info

// Hero info
string name = hero.info.ID;
Sprite icon = hero.info.Icon;
string desc = hero.info.Description;

// Key-value data with fallbacks
int level = hero.info.GetInt("level", 1);
float speed = hero.info.GetFloat("speed", 5f);
string faction = hero.info.GetString("faction", "neutral");

// Check if a key exists
if (hero.info.Has("faction")) { ... }

Finding extensions by info ID

// By type (existing)
MyAbility ability = hero.GetExtension<MyAbility>();

// By info ID (new)
AFPCExtension grapple = hero.GetExtension("grapple");
float range = grapple.info.GetFloat("range", 20f);
Inspector The Info section appears as a collapsible toggle in the inspector for Hero and all extensions. It is drawn automatically by the base editor class — no custom editor code needed in your extensions.

Custom MonoBehaviours

You do not need any special base class to react to AFPC events. Any MonoBehaviour can subscribe to events on hero.lifecycle, hero.movement, and hero.overview.

using UnityEngine;
using AFPC;

public class HealthDisplay : MonoBehaviour {

    [SerializeField] private Hero hero;
    [SerializeField] private UnityEngine.UI.Slider healthBar;

    private void Start () {
        hero.lifecycle.onHealthChanged.AddListener(UpdateBar);
        UpdateBar(hero.lifecycle.GetHealthValue()); // set initial value
    }

    private void OnDestroy () {
        hero.lifecycle.onHealthChanged.RemoveListener(UpdateBar);
    }

    private void UpdateBar (float current) {
        healthBar.value = current / hero.lifecycle.referenceHealth;
    }
}

You can also assign callbacks directly in the inspector via the UnityEvent fields on Lifecycle, Movement, and Overview — no code required for simple reactions like playing a sound or enabling a GameObject.

Gotchas

Forgetting to unsubscribe from events

If your MonoBehaviour subscribes to a hero event in Start() and does not unsubscribe in OnDestroy(), the callback will fire even after your object is destroyed — causing a NullReferenceException. Always pair AddListener with RemoveListener using a named method, not a lambda.

Reading subsystem state while the character is dead

When lifecycle.Availability() is false, Hero stops updating its subsystems. Values like IsGrounded() or GetEnduranceValue() will be stale. Guard your own scripts with an availability check if the character can die.

private void Update () {
    if (!hero.lifecycle.Availability()) return;
    // safe to read movement and overview state here
}

Ground Mask includes the wrong layers

If the character floats, bounces, or never registers landing: check that Ground Mask includes only the geometry layers and excludes the character's own layer. Including the character's own collider in the ground check causes it to detect itself.

Ground detection on modular geometry

If the character jitters or loses ground on floors built from tiled/modular pieces, switch Ground Detection from SphereCast to MultiRaycast (or Raycast). SphereCast can catch on the vertical faces at seams between adjacent colliders. Raycast-based modes cast downward only and are immune to this issue.

Camera shake requires Physical Camera

overview.Shake() works by offsetting the camera's Lens Shift, which is a Physical Camera property. Select the Camera component and enable Physical Camera in the inspector, otherwise shake has no visible effect.

Deactivate vs BanMovement

lifecycle.Deactivate() stops everything — movement, look, recovery, all subsystem updates. movement.BanMovement() stops only walking/running while leaving the camera and all other systems running. Use Deactivate for death and cutscenes; use the Ban methods for partial locks like dialogue or scoped restrictions.

Sensitivity set to zero

If the camera does not move, check overview.sensitivity. The Wizard flags this as an error, but if you set it at runtime make sure it is greater than zero.
